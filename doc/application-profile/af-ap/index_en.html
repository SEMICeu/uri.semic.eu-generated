Summary

This is an application profile, aimed to extend the use of the Alignment Format, originally envisaged for the description of an alignment of Core Vocabularies with other vocabularies.

Status of this document

This specification has the status of draft published on 2023-02-08.

Overview

This document describes the usage of the following entities for a correct usage of the Application Profile:
| Alignment | Cell |

Entities

Alignment

Definition
It describes a particular alignment.
Properties
For this entity the following properties are defined: created, description, license, map, modified, publisher, source ontology, source version, target ontology, target version, title.
Property Expected Range Cardinality Definition Usage Codelist
created date 1 Date of creation of the alignment
description Literal 1..* A free-text account of the aligment.
license Resource 1..* A legal document giving official permission to do something with the alignment
map Cell 0..* A correspondance between entities of the ontologies.
modified date 0..1 Date of last change of the alignment
publisher Resource 1..* An entity responsible for making the alignment available.
source ontology Resource 1 The URI of the first aligned ontology.
source version string 1 Version IRI or version string of the source ontology.
target ontology Resource 1 The URI of the second aligned ontology.
target version string 1 Version IRI or version string of the source ontology.
title Literal 1 A name given to the alignment.

Cell

Definition
A correspondence between 2 entities in an alignment.
Properties
For this entity the following properties are defined: comment, description, mapping confidence, mapping justification, mappingURI, relation, source entity, source identifier, source label, target entity, target identifier, target label.
Property Expected Range Cardinality Definition Usage Codelist
comment Literal 0..1 Free text field containing either curator notes or text generated by tool providing additional informative information.
description Literal 0..1 A description of the mapping.
mapping confidence float 1 A confidence of the mapping relation.
mapping justification Resource 1 A mapping justification is an action (or the written representation of that action) of showing a mapping to be right or reasonable.
mappingURI Resource 1 The URI of the relation between the source Entity to the target Entity.
relation string 1 The relation holding between the two entities.
source entity Resource 1 The URI of the first aligned ontology entity.
source identifier string 0..1 The unique identifier of the source entity. Values, for example, could be an Xpath of the entity contained inside an XML message.
source label Literal 0..1 The label of the source entity of the mapping.
target entity Resource 1 The URI of the second aligned ontology entity.
target identifier string 0..1 The unique identifier of the target entity. Values, for example, could be an Xpath of the entity contained inside an XML message.
target label Literal 0..1 The label of the target entity of the mapping.
