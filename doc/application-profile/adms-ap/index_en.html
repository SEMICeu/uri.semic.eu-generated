Summary

Locations can be described in three principal ways: by using a place name, a geometry or an address. The specific context will determine which method of describing a location is most appropriate. The Core Location Vocabulary provides structure for all three.

ISO 19112 defines a location as "an identifiable geographic place." With this in mind, "Eiffel Tower", "Madrid" and "California" are all locations and this is a common way of representing locations in public sector data, i.e. simply by using a recognised name. Such identifiers are common although they can be highly ambiguous as many places share the same or similar names.

In addition to a simple (string) label or name for a Location, this vocabulary defines a property that allows a Location to be defined by a URI, such as a GeoNames or DBpedia URI.

No cardinality constraints are placed on any property of the Location, Address or Geometry classes in order to maximise flexibility. A single address may be defined in different ways, a geometry may be defined using different coordinate reference systems and a single place may have no recognised name or multiple names. The Core Location Vocabulary makes a minimum number of assumptions about what data will be encoded. However, it clearly makes no sense to define any of the location classes without any properties or to provide multiple instances of the same property with conflicting values.

Entities

Asset

Description
Abstract entity that reflects the intellectual content of an Asset and represents those characteristics that are independent of its physical embodiment.
Usage
This abstract entity combines the FRBR entities work (a distinct intellectual or artistic creation) and expression (the intellectual or artistic realization of a work). The physical embodiment of an Asset is called an Asset Distribution. A particular Asset may have zero or more Distributions.
Properties
For this entity the following properties are defined: alternative name, contact point, description, distribution, has version, identifier, included asset, is version of, issued, keyword, landing page, language, modified, page, publisher, relation, spatial, status, theme, title, translation, type, version info, version notes.
Property Expected Range Description Usage Codelist
alternative name Literal Alternative name for the Asset.
contact point Contact Information Contact point for further information about the Asset, where errors can be reported or suggestions can be made.
description Literal Descriptive text for the Asset.
distribution Asset Distribution Implementation of the Asset in a particular format .
has version Asset An Asset that is the solution of which this Asset is a release.
identifier Literal Main identifier for the Asset, e.g. the URI or other unique identifier in the context of the Repository.
included asset Asset An Asset that is contained in the Asset being described .
is version of Asset An Asset that is the solution of which this Asset is a release.
issued Literal Date of formal issuance of this version of the Asset.
keyword Literal Word of phrase to describe the Asset.
landing page Documentation Web page that provides access to the asset, its releases, distributions and/or additional information. It is intended to point to a landing page at the original asset provider, not to a page on a site of a third party, such as a catalogue or repository.
language Language Language of the Asset, or language supported by software, using a controlled vocabulary. The recommended vocabulary is the modified Literal Date of latest update of Asset.
page Documentation Documentation that contains information related to the asset.
publisher Publisher Organisation making the Asset available. The publisher is the Agent that publishes the Asset, not the Agent that publishes the metadata about it.
relation Resource Related resource, in particular a related asset.
spatial Geographical Coverage Geographic region or jurisdiction to which the Asset applies, using a controlled vocabulary. The recommended controlled vocabulary are the MDR Places Named Authority List from the Publications Office.
status Asset Type Status of the Asset in the context of a particular workflow process, using a controlled vocabulary. The recommended controlled vocabulary is the ADMS Status vocabulary.
theme Asset Type Theme or sector to which the Asset applies, using a controlled vocabulary. The recommended controlled vocabulary is Dataset Theme Vocabulary from Publications Office.
title Literal Name of the Asset.
translation Asset Translation of the Asset.
type Asset Type Type of the Asset, using a controlled vocabulary. It is recommended to use the EIRA vocabulary.
version info Literal Version number or other designation of the Asset.
version notes Literal Description of changes between this version and the previous version of the Asset.

Asset Distribution

Description
Particular physical embodiment of an Asset.
Usage
It is an example of the FRBR entity manifestation (the physical embodiment of an expression of a work). A Distribution is typically a downloadable computer file (but in principle it could also be a paper document or API response) that implements the intellectual content of an Asset. A particular Distribution is associated with one and only one Asset, while all Distributions of an Asset share the same intellectual content in different physical formats.
Properties
For this entity the following properties are defined: access URL, checksum, description, file size, format, issued, license, modified, publisher, status, title.
Property Expected Range Description Usage Codelist
access URL Resource URL of the Distribution
checksum Checksum Checksum of the distribution.
description Literal Descriptive text for the Distribution.
file size Literal Size of the file of the distribution.
format Media Type Media type of the Distribution, using a controlled vocabulary.
issued Literal Date of formal issuance of the Distribution.
license Licence Conditions or restrictions for (re)use of the Distribution. if there are multiple licences, the most 'open' one should be given. Whenever relevant, the URIs under the concept scheme identified by http://purl.org/adms/OSIlicence/ can be used.
modified Literal Date of latest update of the Distribution.
publisher Publisher Organisation making the Distribution available. The publisher is the Agent that publishes the Distribution, not the Agent that publishes the metadata about it.
status Asset Type Status of the Distribution in the context of a particular workflow process, using a controlled vocabulary The recommended controlled vocabulary is the ADMS Status vocabulary
title Literal Name of the Distribution.

Asset Repository

Description
System or service that provides facilities for storage and maintenance of descriptions of Assets and Asset Distributions, and functionality that allows users to search and access these descriptions.
Usage
An Asset Repository will typically contain descriptions of several Assets and related Asset Distributions.
Properties
For this entity the following properties are defined: contact point, dataset, description, homepage, issued, keyword, publisher, spatial, theme taxonomy, title.
Property Expected Range Description Usage Codelist
contact point Contact Information Contact point for further information about the Repository.
dataset Asset An Asset included in the Repositor.
description Literal Descriptive text for the Repository.
homepage Documentation Web page that gives access to the Repository.
issued Literal Date of formal issuance of the Repository.
keyword Literal Word of phrase to describe the Repository.
publisher Publisher Organisation making the Repository available. The publisher is the Agent that publishes the Repository, not the Agent that publishes the metadata about it.
spatial Geographical Coverage Geographic region or jurisdiction to which the Repository applies, using a controlled vocabulary. The recommended controlled vocabulary are the MDR Places Named Authority List from the Publications Office.
theme taxonomy Theme Taxonomy Concept Scheme used to classify an Asset Repository's assets, using a controlled vocabulary. The recommended controlled vocabulary is the Dataset Theme Vocabulary. The value to be used for this property is the URI of the vocabulary itself, i.e. the concept scheme, not the URIs of the concepts in the vocabulary.
title Literal Name of the Repository.
