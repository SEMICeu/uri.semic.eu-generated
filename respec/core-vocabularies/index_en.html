Introduction

This document describes an vocabulary, in this case Core Vocabularies. A vocabulary is a semantic data specification designed so that the included terms are highly reusable. The included classes and properties come with a limited set of constraints.

This allows them to be used in a broad range of use cases. This document focuses on ADMS, which is a profile of DCAT used to describe semantic assets, such as metadata and reference data. ADMS is designed for those searching for a document-like asset that can be easily opened and read using familiar desktop software, rather than a dataset that needs to be processed. Although there are overlaps between assets and datasets, the difference in user expectation is at the core of what distinguishes ADMS from DCAT. Moreover, while DCAT is centred on facilitating interoperability between data catalogues, ADMS focuses on the assets within a catalogue.

The Core Business Vocabulary provides a minimum set of classes and properties for legal entities, that is, trading bodies that are formally registered with the relevant authority and that appear in business registers. This excludes sole traders, virtual organisations and other types of 'agents' that are able to do business.

Conformance

Regarding conformity, all the authoring guidelines, diagrams, examples, and notes in this specification are not normative, along with sections marked as non-normative. Everything else is normative, and the key words MUST, MUST NOT, REQUIRED, SHOULD, SHOULD NOT, RECOMMENDED, MAY, and OPTIONAL are interpreted as defined in [rfc2119].
