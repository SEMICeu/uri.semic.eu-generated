Summary

This document describes the Regulatory Reporting Metadata Vocabulary that was developed in the context of the SORTIS initiative. The purpose of this vocabulary is to provide the minimum necessary concepts to describe a request, originating from a legal resource, the resulting action and several key concepts related to its execution.

Status of this document

This Core Vocabulary has the status of Semic Recommendation published on 2024-05-31.

Overview

This document describes the usage of the following entities for a correct usage of the Core Vocabulary:
| Action | Action Result | Agent | Agent Role | Agent Type | Expression | Frequency | Manifestation | Organization | Period of Time | Person | Request | Result Type | Role | Role Type | Status | Temporal Function | Work |

Entities

Action

Description
An event with at least one agent that is participant in it in a given time.
Properties
For this entity the following properties are defined: at time, has agent role, has annotation, has next, has result, has status, related to.
Property Expected Range Description Usage Codelist
at time Period of Time An Action or a Role is valid within a time interval.
has agent role Agent Role This property connects the executed Action to the Agent that was involved in its exectuion.
has annotation xsd:string Annotation related to the Action or Request made by the domain expert.
has next Action A property linking an Action with the succeeding Action in a repetitive sequence of actions generated by a Request.
has result Action Result A property that links an Action with the Result of this action.
has status Status A property that links an Action with the Status of this action.
related to Action Indicates a somehow related other document, not necessarily a legal resource.

Action Result

Description
The Result of an Action. It could be an activity (e.g., to do something) or an output (e.g., report).
Properties
For this entity the following properties are defined: has document, has title, has type.
Property Expected Range Description Usage Codelist
has document Expression A property that links a Result of an Action with an Expression, usually wen the result is a document.
has title xsd:string Title of the Action.
has type Result Type This property connects the Action Result to the Action Result Type to which it belongs.

Expression

Description
The intellectual realisation of a Work in the form of identifiable immaterial objects, such as, but not limited to, text.
Properties
For this entity the following properties are defined: has URI, is realized by.
Property Expected Range Description Usage Codelist
has URI xsd:anyURI URI of the Legal Source Expression where the Request is expressed.
is realized by Work Relates a work to an expression of this work in the form of a "sequence of signs" (typically alpha-numeric characters in a legal context).

Period of Time

Description
A temporal entity with non-zero extent or duration.
Usage
The value of the start time and end time properties should be different,and/or a duration should be given.
Properties
For this entity the following properties are defined: has function, is incomplete.
Property Expected Range Description Usage Codelist
has function Temporal Function A property that links one Period of Time with its Function inside of the Request (e.g., deadeline, term, enter into force).
is incomplete xsd:boolean Data property that track the incompleteness of the temporal parameters.

Request

Description
A portion of legal source expression including a mandate that produces at least one Action.
Properties
For this entity the following properties are defined: changes, cites, cites, has annotation, has topic, has unit, has URI, has value, is part of, produces, related to.
Property Expected Range Description Usage Codelist
changes Request Indicates that this work or expression legally changes another. This encompasses the notions of amendment, replacement, repeal, or other types of change. This may be a direct change (textual or non-textual amendment) or a consequential or indirect change.
cites Expression Citation in the text of the legislation. This may be at the legal resource or legal expression level, as required by the implementation context. This includes verbatim citation and citations in referrals.
cites Request Citation in the text of the legislation. This may be at the legal resource or legal expression level, as required by the implementation context. This includes verbatim citation and citations in referrals.
has annotation xsd:string Annotation related to the Action or Request made by the domain expert.
has topic skos:Concept The topic of the request.
has unit Frequency The property linking a Request with the Frequency and expressing the unit of this frequency.
has URI xsd:anyURI URI of the Request.
has value Frequency The property linking a Request a Frequency expressing the value of the unit of the frequency (e.g., 2 is the value, quarter is the unit).
is part of Expression Indicates a work in which this one is physically included. Covers the case of text included in an Official Journal, or an article included in a text.
produces Action A Request produces one or many Action.
related to Request Indicates a somehow related other document, not necessarily a legal resource.
