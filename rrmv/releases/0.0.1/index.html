Summary

This document describes the Regulatory Reporting Metadata Vocabulary that was developed in the context of the SORTIS initiative. The purpose of this vocabulary is to provide the minimum necessary concepts to describe a request, originating from a legal resource, the resulting action and several key concepts related to its execution.

Status of this document

This Core Vocabulary has the status of Semic Recommendation published on 2024-05-31.

Overview

This document describes the usage of the following entities for a correct usage of the Core Vocabulary:

Entities

Action

Description
An event with at least one agent that is participant in it in a given time.
Properties
For this entity the following properties are defined: at time, frequency, has result, involves role, related to.
Property Expected Range Description Usage Codelist
at time Time instant The time at which an action takes place or has to take place.
frequency skos:Concept The frequency of an Request over time (e.g., every 6 months, each quarter).
has result Action Result A property that links an Action with the Result of this action.
involves role for kind The property linking an Action to Role involved in the Action.
related to Action Indicates a somehow related other document, not necessarily a legal resource.

Action Result

Description
The Result of an Action. It could be an activity (e.g., to do something) or an output (e.g., report).
Properties
For this entity the following properties are defined: described in, has type.
Property Expected Range Description Usage Codelist
described in Legal Resource This property connects an Action Result to the Legal Resource in which it is described.
has type Action Result Type This property connects the Action Result to the Action Result Type to which it belongs.

Description

Exectution Action Result

Description
The Result from the execution of an Action.
Properties
For this entity the following properties are defined: accepted at, conforms to, delivered at.
Property Expected Range Description Usage Codelist
accepted at Time instant The time at which the Result of an executed Action is or was accepted.
conforms to Action Result An established standard to which the described resource conforms.
delivered at Time instant The time at which the Result of an executed Action is or was delivered.

Execution Action

Description
The exection of the Action.
Properties
For this entity the following properties are defined: executed at time, had result, has agent role.
Property Expected Range Description Usage Codelist
executed at time Time instant The time at which the action is exectued.
had result Exectution Action Result The result of the executed Action.
has agent role Agent Role This property connects the executed Action to the Agent that was involved in its exectuion.

Execution Request

Description
A portion of legal source expression including a mandate that produces at least one Action that is being executed.
Properties
For this entity the following properties are defined: initiated at, initiates, is execution of.
Property Expected Range Description Usage Codelist
initiated at Time instant The time at which the Execution of the Request is or was initiated.
initiates Execution Action This property links an Execution Request with an Execution Action.
is execution of Request This property links the Execution Request with the Request.

for kind

Description
The kind of role of the agent as is described in the legal resource.
Properties
For this entity the following properties are defined: for role, for role.
Property Expected Range Description Usage Codelist
for role Role This property links the kind of a Role to a Role.
for role Agent Kind This property links the kind of a Role with the kind of an Agent.

Legal Resource

Description
This class represents the legislation,policy or policies that lie behind the Rules that govern the service.
Usage
The definition and properties of the Legal Resource class are aligned with the ontology included in "Council conclusions inviting the introduction of the European Legislation Identifier (ELI)". For describing the attributes of a Legal Resource (labels, preferred labels, alternative labels, definition, etc.) we refer to the ELI ontology. In this data specification the use is restricted to instances of this class that follow the ELI URI guidelines.
Properties
For this entity the following properties are defined: has URI.
Property Expected Range Description Usage Codelist
has URI rdfs:Resource URI of the Legal Source Expression where the Request is expressed.

Request

Description
A portion of legal source expression including a mandate that produces at least one Action.
Properties
For this entity the following properties are defined: derived from, produces, related to, topic.
Property Expected Range Description Usage Codelist
derived from Legal Resource The legal resource or legal document from which the Request is derived.
produces Action A Request produces one or many Action.
related to Request Indicates a somehow related other document, not necessarily a legal resource.
topic skos:Concept The topic of the request.

Role

Description
Role is played by and Agent involved in the Request.
Properties
No properties have been defined for this entity.
